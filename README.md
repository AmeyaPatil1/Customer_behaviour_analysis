Customer Shopping Behaviour Analysis
Overview

This project analyzes customer shopping behavior using Python, SQL, and Power BI. The goal is to extract insights from raw data, clean and transform it, run analytical queries on a PostgreSQL database, and present the results in an interactive dashboard.

The project follows a complete data analytics workflow from ingestion to visualization.

Dataset

The dataset contains customer shopping transactions including:

Customer demographics

Purchase history

Product categories

Spending patterns

It is loaded and explored using Python before being stored and queried in a PostgreSQL database.

Tools & Technologies

Python – Data loading, EDA, and cleaning

Pandas, NumPy, Matplotlib, Seaborn – Data analysis and visualization

PostgreSQL – Data storage and SQL querying

SQL – Analytical queries

Power BI – Dashboard and reporting

Project Steps

Data Loading

Load dataset using Python and Pandas

Exploratory Data Analysis (EDA)

Understand data structure

Analyze distributions and trends

Identify missing values and anomalies

Data Cleaning

Handle missing values

Fix data types

Remove duplicates

Standardize columns

Database Integration

Load cleaned data into PostgreSQL

Create tables and relationships

SQL Analysis

Run analytical queries to extract insights

Generate metrics for dashboard

Dashboard Development

Build interactive Power BI dashboard

Visualize KPIs and trends

Dashboard

The Power BI dashboard provides:

Customer segmentation

Sales and revenue trends

Category-wise performance

Spending behavior analysis

It enables business users to explore data and make data-driven decisions.

Results

Key outcomes include:

Identification of top-performing product categories

Customer spending patterns by segment

Purchase frequency and revenue trends

Actionable insights for marketing and sales strategy
